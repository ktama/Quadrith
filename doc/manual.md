# Quadrith ユーザーマニュアル

3軸（重要度・緊急度・状態）でタスクを管理するデスクトップアプリ **Quadrith** の使い方ガイドです。

---

## 1. Quadrith でできること

- **重要度 × 緊急度のマトリクス**にタスクを置いて「どれからやるか」を一目で把握できます。
- 第3の軸である**状態**（未着手・進行中・保留・待ち・完了）は、色とフィルタで表現します。
- **Today（フォーカスビュー）**で「今日やること」だけに絞って実行できます（§4.1）。
- 各タスクに**工数（S/M/L/XL）**の見積りを持たせ、1日の容量や統計を時間で見られます（§3.10）。
- カンバン・アーカイブ・統計など、同じデータを別の切り口で見られます。**週次レビュー**で1週間を振り返れます（§4.2）。
- データはすべて**手元（ローカル）に保存**され、外部に送信されません。

> 💡 **考え方のヒント**: 右上「今すぐやる」だけに追われていないかを、たまに見直しましょう。
> 本当に効くのは左上「計画する」（重要だけど急ぎではない）領域です。Quadrith はこの領域の
> 放置を**リマインド**で教えてくれます。

---

## 2. 画面の基本構成

```
┌───────────────────────────────────────────────────────┐
│ [ロゴ] Quadrith [Today|マトリクス|カンバン|繰り返し|アーカイブ|統計] 🔍 🔔 ⚙ ─ ▢ ✕ │ ← ヘッダー(タイトルバー兼用)
│ [未着手][進行中][保留][待ち][完了]  タグ:[仕事][個人]      │ ← フィルタ行
├───────────────────────────────────────────┬───────────┤
│                                           │           │
│            マトリクス(4象限)               │  詳細パネル │
│                                           │ (カード選択時)│
│                                           │           │
├───────────────────────────────────────────┘           │
│ インボックス(未仕分け) [+ タスクを追加してEnter]          │
└───────────────────────────────────────────────────────┘
```

- **ヘッダー**: アプリのタイトルバーを兼ねています。左にロゴとビュー切替タブ、右に検索・リマインド（🔔）・
  設定（⚙）・ウィンドウ操作（最小化／最大化／閉じる）。
- **フィルタ行**: 状態チップ・タグ・の絞り込み（マトリクス／カンバン／アーカイブ表示時のみ）。
- **マトリクス**: 縦軸＝重要度（上が高い）、横軸＝緊急度（右が高い）。4つの象限にうっすらラベル。
- **インボックス**: まだ仕分けていないタスクの置き場（画面下部）。
- **詳細パネル**: カードを選ぶと右側に開きます。

## 3. タスクの基本操作

### 3.1 追加する

- **インボックスに追加**: 画面下の入力欄にタイトルを入れて **Enter**。未仕分け（座標なし・未着手）で入ります。
- **クイック追加（どの画面でも）**: グローバルホットキー **Ctrl+Shift+Space**（初期値）で小さな入力窓が
  出ます。タイトルを入れて Enter → インボックスへ。Esc で閉じます。

### 3.2 マトリクスに配置する（仕分け）

- インボックスのカードを**マトリクスへドラッグ**すると、その位置の重要度・緊急度が保存されます。
- 逆に、マトリクスのカードを**インボックスへドラッグ**すると座標が外れて差し戻されます
  （カードを右クリック →「インボックスへ戻す」でも可）。
- カードを**少しだけ動かしてクリック**すると、ドラッグではなく**詳細パネル**が開きます。

### 3.3 詳細を編集する

カードをクリックして右の詳細パネルで編集します。

| 項目       | 操作                                                                              |
| ---------- | --------------------------------------------------------------------------------- |
| タイトル   | クリックして編集。Enter または欄外クリックで確定                                  |
| 状態       | チップをクリックして変更（未着手／進行中／保留／待ち／完了）                      |
| 期限日     | カレンダーから選択。期限が近い／過ぎたカードはバッジが警告色に                    |
| 再確認日   | 状態を「保留」「待ち」にすると入力欄が出ます（後述のリマインド用）                |
| メモ       | **Markdown 対応**。「編集／プレビュー」を切り替え。リンクは外部ブラウザで開きます |
| 工数       | **S／M／L／XL**（または未見積り）。所要時間の概算（§3.10）                        |
| 今日やる   | 「☆ 今日やるに追加」で **Today** に入れる／「★ 解除」で外す（§4.1）              |
| カテゴリ   | 設定で登録した候補から選択（Redmine エクスポート用、§11）。任意                  |
| タグ       | 既存タグをクリックで付け外し。下の欄＋色で新規作成して追加                        |
| 完了／削除 | 「完了にする」（緑）／「削除」                                                    |

### 3.4 状態を変える方法は3通り

1. **詳細パネル**の状態チップ
2. **カードを右クリック** → メニューから状態を選ぶ
3. **カンバンビュー**で列から列へドラッグ

### 3.5 完了とアーカイブ

- 状態を「完了」にすると、**既定で24時間後**にマトリクスから消えて**アーカイブ**へ移ります
  （消えるまでの時間は設定で変更可）。
- アーカイブされたタスクは「アーカイブ」ビューでいつでも見られ、「復元」で戻せます。

### 3.6 削除・元に戻す・ごみ箱

- カードを削除すると**「元に戻す」付きのトースト**が5秒間出ます。押せばすぐ復活します。
- 削除は**論理削除**なので、見えなくなっても「アーカイブ」→「ごみ箱」タブから復元できます。
- ごみ箱のタスクは**30日後に自動で完全削除**されます。すぐ消したいときは「完全削除」（確認あり）。

### 3.7 繰り返し（定期タスク）

「毎週ゴミ出し」「毎月の家賃支払い」のように**定期的に発生するタスク**は、毎回手で作る代わりに
**ひな型**を登録しておくと、発生日が来たときに自動でタスクが作られます。

**登録のしかた（2通り）**

1. **既存のタスクから**: 詳細パネルを開き、右上の **🔁** ボタン → モーダルで頻度を設定して「登録」。
   そのタスクの内容・配置（象限）・タグを引き継いだ繰り返しになります。
2. **ゼロから**: 「繰り返し」ビューの右上 **「＋ 新規作成」** → タイトルと頻度を入力して保存。

**設定できる頻度**

| 種類 | 例                                           |
| ---- | -------------------------------------------- |
| 毎日 | 毎日 / 3日ごと                               |
| 毎週 | 毎週 月・水・金 / 隔週 月曜（曜日は複数可）  |
| 毎月 | 毎月15日 / 2ヶ月ごと（その日が無い月は末日） |
| 毎年 | 毎年4月1日                                   |

**動きかた**

- 発生日になると、ひな型と**同じ象限・タグ・カテゴリ**でタスクが「未着手」で作られます（カードに **🔁** が付きます）。
- 生成されたタスクは**普通のタスク**です。完了・編集・通知・統計・アーカイブはこれまでどおり。
- アプリを数日起動しなかった間に複数回の発生日が過ぎても、**まとめて1件**だけ作られます（溜まりません）。
- 前回ぶんがまだ終わっていない（未完了の）うちは、次のぶんは作られません。
- 既存タスクから登録した場合、そのタスク自身が当日ぶんを兼ねるため、生成は**次回の発生日から**始まります。

**管理（「繰り返し」ビュー）**

- 各ひな型の頻度と**次回発生日**が一覧で見られます。
- **停止 / 再開**: 一時的に生成を止める・再開する（ひな型は残ります）。
- **編集**: 頻度や内容を変更（次回生成ぶんから反映されます）。
- **削除**: その繰り返しを終了します。**すでに生成済みのタスクは残ります**。

### 3.8 複数のタスクをまとめて操作する

たくさんのカードを一度に片付けたいときは、**複数選択**して一括操作できます。

- **Ctrl（Mac は ⌘）を押しながらカードをクリック**で、選択に追加・解除します（マトリクス・カンバン共通）。
- **2件以上**選ぶと、右側の詳細パネルが**一括操作バー**に変わります。ここから:
  - **状態をまとめて変更**
  - **工数（S/M/L/XL・未見積り）をまとめて設定**
  - **「今日やる」にまとめて追加／解除**
  - **タグをまとめて付ける／外す**（各タグが選択中の何件に付いているかも表示）
  - **まとめてインボックスへ戻す**
  - **まとめて削除**（「元に戻す」は**1回で全部復活**します）
- **まとめて移動**: 選択したカードの1枚を（Ctrl なしで）マトリクス上でドラッグすると、選択した
  カードが**相対位置を保ったまま一緒に移動**します。インボックスへドロップすれば全部差し戻します。
- **右クリックも選択に連動**: 選択中のカードを右クリックすると、メニューの操作（状態変更・インボックスへ・
  削除）が**選択全体に適用**されます（選択外のカードを右クリックした場合は従来どおりその1件だけ）。
- **Esc** で選択を解除します。ふつうのクリック（Ctrl なし）に戻ると単一選択に戻ります。

### 3.9 コマンドパレットですばやく操作する（Ctrl+K）

**Ctrl+K（Mac は ⌘K）** でコマンドパレットが開きます。キーボードだけで素早く操作できます。

- **ビューの移動**（マトリクス／カンバン／繰り返し／アーカイブ／統計／設定）
- **選択中タスクへの操作**（状態変更・タグ・インボックスへ・削除）— 上の複数選択と組み合わせられます
- **タスクの検索ジャンプ**（タイトルで絞り込み → 選ぶと詳細が開く）
- **新規作成**（入力した文字をそのままインボックスに追加）

入力して候補を絞り込み、**↑↓** で移動、**Enter** で実行、**Esc** で閉じます。

- **ビューの移動**には「Today」も加わりました。
- **選択中タスクへの操作**に「今日やるに追加／解除」、単独コマンドとして「**週次レビューを開始**」も選べます。

### 3.10 工数（所要時間の見積り）

タスクに「だいたいどれくらいかかるか」を **Tシャツサイズ（S／M／L／XL）** で持たせられます。
入力の速さを優先したざっくり見積りです。

| サイズ | 目安の分（既定） | イメージ   |
| ------ | ---------------- | ---------- |
| S      | 15分             | ちょい作業 |
| M      | 60分             | 1時間級    |
| L      | 180分            | 半日級     |
| XL     | 480分            | 1日級      |

- **付け方**: 詳細パネルの「工数」、または複数選択中の一括操作バーから。繰り返しのひな型にも設定でき、
  生成されるタスクへ引き継がれます。
- **未見積り**でもOK（容量・時間の集計には数えず「未見積り」として別に件数表示します）。
- マトリクスのカードには工数が**バッジ**で出ます（カードの大きさは変わりません）。
- サイズ→分の対応は**設定で変更**できます（§10「Today / 工数 / レビュー」）。変えると Today の容量
  メーターや統計の時間集計もそれに追従します。

## 4. ビューの使い分け

ヘッダーのタブで切り替えます。

| ビュー         | 用途                                                              |
| -------------- | ----------------------------------------------------------------- |
| **Today**      | 本日やることに集中。締切・進行中（自動）＋「今日やる」（選択）（§4.1） |
| **マトリクス** | メイン。重要度×緊急度でタスクを配置・俯瞰                         |
| **カンバン**   | 状態別の列。列間ドラッグで状態を変えながら進捗管理                |
| **繰り返し**   | 定期タスクのひな型を管理（一覧・新規作成・停止/再開・削除、§3.7） |
| **アーカイブ** | 完了タスクの一覧（復元可）＋ ごみ箱タブ                           |
| **統計**       | 象限分布＋深掘り指標＋**週次レビュー**。JSON/CSV・Redmine エクスポートもここから（§4.2） |

> **使い分けのイメージ**: マトリクス＝計画、カンバン＝進捗、**Today＝本日の実行**。

### 4.1 Today（フォーカスビュー）

「今日やること」だけを集めた実行用のビューです。2つのグループに分かれます。

- **締切・進行中（自動）**: ほうっておいても*出てくる*もの。**理由バッジ**が付きます。
  - 🔴 期限超過 / 🟠 今日期限 / 🔵 進行中 / 🟣 再確認 /（設定で）⚡ 今すぐ（「今すぐやる」象限）
  - 期限が古い順 → 今日期限 → 進行中 → 再確認 の順に並びます。手では外せません（完了・期限変更・状態変更で自然に消えます）。
- **今日やる（選択）**: 自分で*選んで*入れたもの。詳細パネルや一括操作、コマンドパレットから「今日やる」に追加します。
  - 並び順は **▲▼ ボタン**で手動調整でき、保存されます。
  - 自動グループにも該当するタスクは、自動側に「★ 今日やる」バッジが付き、二重には出ません。

**容量メーター**: 画面上部に「見積り ◯分 / 可処分 ◯分」が出ます。未完了タスクの工数（§3.10）の合計で、
可処分（既定360分・設定で変更可）を超えると赤で警告します。未見積りのタスクは合計に入れず件数だけ併記します。

**繰り越し**: 「今日やる」にしたまま終わらなかったタスクは、**翌日（アプリ起動時・日付が変わったとき）に
自動で当日へ持ち越し**ます（並び順は保たれます）。完了したものは持ち越さず、その日の記録として残ります
（週次レビューの「計画遵守率」に使われます）。

その日の「今日やる」が空のときは、画面が**「計画する」象限から引き込む**導線（候補一覧）を出します。

### 4.2 統計の深掘りと週次レビュー

「統計」ビューでは、完了タスクの象限分布に加えて次の**深掘り指標**が見られます。

- 週ごとの**完了スループット**（件数＋工数の分）と**作成 vs 完了**（流入に追いつけているか）
- **第2領域リードタイム**（重要×非緊急が完了まで掛かる日数の中央値）、**未見積り率**、**計画遵守率**
- **最古の未着手**・**第2領域の放置**の上位

**週次レビュー**（「統計」右上の「週次レビュー」ボタン、またはコマンドパレットから開始）は、1週間を
振り返りながら**その場で手を動かせる**6ステップのウィザードです。

1. 今週の振り返り（完了量・先週比・計画遵守率・象限バランス）
2. 期限の棚卸し（期限変更／完了／今日やる追加）
3. 第2領域の放置（緊急へ移動／計画／破棄）
4. 保留・待ちの再確認（再確認日の更新／状態変更）
5. インボックス仕分け（象限へ割当／削除）
6. 来週の計画（「今日やる」を仕込む）

- 「次へ／戻る／スキップ」で進み、最後に**前回レビュー日時**を記録します。
- **毎週リマインド**（既定オフ）を有効にすると、指定の曜日以降にアプリ内バナーで知らせます
  （§10「Today / 工数 / レビュー」）。

## 5. 絞り込みと検索

- **状態フィルタ**: ヘッダーの状態チップをクリックでオン/オフ（複数選択可）。
- **タグ絞り込み**: タグチップをクリック。選んだタグのいずれかを持つタスクだけ表示。
- **検索**: 右上の検索ボックスにキーワード。**タイトルとメモ**を対象に絞り込みます。

> これらの絞り込みはマトリクス／カンバン／アーカイブに同時に効きます（統計・設定には影響しません）。

## 6. リマインド（🔔）

ヘッダーのベルに、対応が必要なタスクの件数が出ます。クリックで一覧、項目を選ぶとそのタスクが開きます。

リマインドは3種類:

| 種類       | 条件                                                              |
| ---------- | ----------------------------------------------------------------- |
| **期限**   | 期限日が今日、または過ぎている（未完了）                          |
| **再確認** | 再確認日が来た「保留」「待ち」のタスク                            |
| **放置**   | 「計画する」象限（重要×非緊急）を一定期間（既定14日）進めていない |

- **通知（Windowsトースト）**は「期限」と「再確認」について、設定した時刻（既定 **9:00**）に発火します。
  「放置」リマインドはアプリ内のベルのみです。
- 「放置」の判定は**状態を変えた日**が基準です。ドラッグやタグ編集ではリセットされません
  （＝本当に手を付けていないタスクだけが残ります）。

## 7. カードが重なったとき（クラスタ表示）

- 同じあたりにカードが重なっても、保存位置は変えずに**描画時だけ少しずらして**重なりを避けます。
- 同一箇所に**4枚以上**密集すると「**+N**」バッジにまとまります。クリックすると一覧の吹き出しが開き、
  選ぶとそのタスクが開きます。

---

## 8. ウィンドウ操作

Quadrith は枠なしウィンドウで、ヘッダー自体がタイトルバーです。

- **移動**: ヘッダーの空いている部分をドラッグ。
- **最大化／元に戻す**: 右上の □ ボタン、またはヘッダーの空き部分をダブルクリック。
- **最小化／閉じる**: 右上の ─ ／ ✕。
- **サイズ変更**: ウィンドウの**端や角をドラッグ**。
- ウィンドウの位置・サイズは次回起動時に**復元**されます（モニタ構成が変わって画面外になった場合は
  自動的に中央へ戻します）。

> **同時起動はできません**。すでに起動中にもう一度開こうとすると、既存のウィンドウが前面に出ます。

## 9. システムトレイ（常駐）

- 既定では、閉じる（✕）を押すと**終了せずトレイへ最小化**します（設定で「終了」に変更可）。
- トレイアイコンを**左クリック**で再表示。**右クリックメニュー**から「開く」「終了」。

## 10. 設定（⚙）

ヘッダー右の歯車から開きます。変更は即時保存されます。

### データ
- **DBファイルの保存先**: 現在のパス表示と「変更」。クラウド同期フォルダ（Dropbox 等）も指定できます（§12）。
- **場所を開く**: DB のあるフォルダをエクスプローラーで表示。
- **自動バックアップの保持世代数**（既定3）と**保存先**（既定は DB と同じフォルダの `backups/`）。
- **今すぐバックアップ**: その場でバックアップを1つ作成します。

### 表示
- **状態ごとの色**: 5状態それぞれの色をカスタマイズ。
- **完了タスクをマトリクスから消すまでの時間**（既定24時間）。
- **テーマ**: ライト／ダーク／システム連動。

### タグ
- タグの**名前変更**（クリック）・**色変更**（左の色見本）・**削除**（全タスクから外れます）。

### カテゴリ
- **カテゴリ候補**の追加・削除。ここで登録した候補を、タスクの詳細パネルや繰り返しの編集画面から
  割り当てられます。カテゴリは Redmine エクスポートの「カテゴリ」列に使われます（タグとは別物です）。

### Redmine エクスポート
- **トラッカー名**: 取り込み先 Redmine に存在する名前に合わせます（不一致だと取り込みが失敗します）。
- **ステータス／優先度の対応**: 本アプリの状態・象限を、Redmine 側の名前へどう変換するか。
- **すべて「新規」で出力する**: Redmine の設定上、新規チケットに他のステータスを付けられない場合に。
- **開始日／カテゴリの列を出力する**: それぞれの列を出すかどうか。

### Today / 工数 / レビュー
- **工数サイズの分換算**: S／M／L／XL に対応する分（既定 15／60／180／480）。Today の容量や統計の時間集計に反映されます。
- **1日の可処分時間**: Today の容量メーターの分母（既定360分）。
- **Today に「今すぐやる」象限を含める**: 重要×緊急のタスクを自動グループに取り込むか（既定オン）。
- **統計の週の起点**: 月曜／日曜（既定 月曜）。週ごとの集計の区切りに使います。
- **週次レビューの毎週リマインド**: オン／オフ（既定オフ）と曜日・時刻。オンにすると、その曜日以降に
  アプリ内バナーで「今週のレビューがまだです」と知らせます。

### 動作
- **クイック追加のホットキー**: 変更すると即座に反映されます（他アプリと競合する場合は変更を）。
- **Windows 起動時に常駐を開始する**（autostart）。
- **閉じるボタンの挙動**: トレイへ最小化 / 終了。
- **通知の発火時刻**: 期限日・再確認日の通知時刻（既定 9:00）。

## 11. データのエクスポート

### 11.1 全データ（JSON / CSV）
- 「統計」ビューの右上から **JSON** または **CSV** で書き出せます。
- 対象は**論理削除分も含む全タスクとタグ**です（バックアップ・移行・表計算での集計などに）。
  **JSON には繰り返しのひな型**も含まれます（CSV は各タスクの表なので `template_id` 列のみ）。
- CSV は Excel で文字化けしないよう UTF-8（BOM 付き）で出力します。

### 11.2 Redmine 取り込み用 CSV
「統計」ビューの「**Redmine エクスポート**」から、Redmine の「チケットのインポート」で取り込める
CSV を書き出せます。

- **期間を指定**して出力します（既定は今日〜30日後）。対象は次の2つです:
  - **期日が期間内の未完了タスク**（期日なしのタスクは対象外）。
  - **期間内に発生する繰り返し**（発生日ごとに1行。すでに作成済みのぶんはそのまま使われます）。
- 列の対応（題名・説明・トラッカー・ステータス・優先度・期日、任意で開始日・カテゴリ）と
  トラッカー名は**設定**で調整できます（§10「Redmine エクスポート」）。
- 重要度×緊急度は Redmine の**優先度**に変換されます。タグは説明文の末尾に補記されます。
- Redmine 側では、取り込み時に列の対応づけを確認してください。ステータスやトラッカーの名前が
  Redmine に存在しないと取り込めない／失敗することがあります。

## 12. バックアップ・データの持ち運び

- **起動時に自動バックアップ**が作成され、設定の世代数だけ保持されます（古いものから削除）。
- **DBファイルをクラウド同期フォルダに置く**と、簡易的なバックアップや複数台での持ち回りができます。
  - 設定 →「DBファイルの保存先」を変更すると、現在のデータを**移動**するか、その場所の**既存データを開く**かを
    選べます。
  - ⚠️ **複数台での同時起動は非対応**です。1台ずつ使ってください（同時に開くとデータが食い違う恐れ）。
- 別PCへ移すときは、DBファイル（`tasks.db`）一式を渡し、移行先で「保存先の変更 →既存を開く」を選びます。

## 14. 困ったときは

**起動したらデータが空になっている／「DBファイルが見つかりません」と出る**
クラウド同期フォルダがまだ同期されていない可能性があります。出てきたダイアログで
「ファイルを探す」で正しい `tasks.db` を指定するか、「既定の場所に戻す」を選んでください。
（同期が終わるのを待ってから開き直すのが安全です。）

**「データベースを開けませんでした」と出る**
DBファイルが壊れた可能性があります。画面に**バックアップの一覧**が出るので、新しいものを選んで
「復元」→「再試行」してください。

**通知が出ない**
- 設定の「通知の発火時刻」を過ぎているか、対象（今日が期限／再確認日）のタスクがあるか確認。
- Windows 側の通知設定（集中モード等）で抑制されていないか確認。
- 「放置」リマインドはトースト通知されません（ベルのみ）。

**クイック追加のホットキーが効かない**
他のアプリが同じキーを使っている可能性があります。設定 →「クイック追加のホットキー」を別の組み合わせに
変更してください。

**ウィンドウのサイズが変えられない／移動できない**
端や角をドラッグでリサイズ、ヘッダーの空き部分をドラッグで移動します。最大化中は一度元のサイズに
戻してから操作してください。

**完了したのにマトリクスから消えない**
完了から「アーカイブまでの時間」（既定24時間）が経つと自動的に消えます。すぐ片付けたい場合は設定で
時間を短くできます。

## 15. 用語

| 用語              | 意味                                                                  |
| ----------------- | --------------------------------------------------------------------- |
| インボックス      | まだ重要度・緊急度を決めていない未仕分けタスクの置き場                |
| 象限              | マトリクスの4区画（今すぐやる／計画する／さばく・任せる／やめる候補） |
| Today             | 「今日やること」に絞った実行用ビュー（自動グループ＋選択グループ）    |
| 今日やる          | タスクを Today の選択グループに入れる印。未完了は翌日へ自動繰り越し   |
| 工数              | タスクの所要時間の概算（S/M/L/XL）。容量・時間統計に使う             |
| 容量メーター      | Today の「見積り合計 / 1日の可処分時間」。超過すると赤で警告          |
| 計画遵守率        | 「今日やる」に入れた日に完了できた割合（週次レビューの指標）          |
| 週次レビュー      | 1週間を振り返り、その場で操作できる6ステップのウィザード             |
| 再確認日          | 「保留」「待ち」のタスクを死蔵させないために通知する日                |
| アーカイブ        | 完了後、一定時間でマトリクスから退避した状態                          |
| ごみ箱            | 論理削除されたタスク（30日で自動完全削除）                            |
| クラスタ（+N）    | 密集したカードをまとめた表示                                          |
| 繰り返し / ひな型 | 定期タスクの設定本体。発生日に通常タスクを自動生成する（🔁）           |
